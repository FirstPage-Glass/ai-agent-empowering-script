# Part 5: Practical Tips 💡

## How to Ask Better Questions 🎯

**1️⃣ Start simple** 🐣
When using it for the first time, start with simple questions to get familiar with how it responds.

**2️⃣ Go deeper step by step** 🧅
Ask the big picture first, then follow up on details. Peel the onion layer by layer.

**3️⃣ Give it a role** 🎭
```
"You are a professional sales assistant, help me write a..."
"You are a marketing consultant, help me analyze..."
```
Give it a role, and it will respond in a tone that fits that role.

**4️⃣ Specify the format** 📋
```
"Use bullet points"
"Present it as a table"
"Explain it in 3 paragraphs"
```

---

## What If You're Not Satisfied? 🔄

1. 🗣️ **Just say so** — "This doesn't work because..." then tell it what to change
2. 📝 **Give an example** — "Like this:..." give it a reference
3. 🧹 **Start over** — Type `/clear` to clear the conversation and ask again
4. ⏪ **Undo changes** — If it modified a file and you're not happy, type `/undo`

> 💡 **Don't be afraid to make mistakes!** OpenCode isn't perfect. If you're not satisfied, just say so — it'll keep improving until it's right.

## Safety Reminders ⚠️

- 🔒 **Don't upload confidential data** — If you're unsure whether data is sensitive, check with your manager first
- ✅ **Verify important information** — AI can sometimes make mistakes. Double-check critical data yourself
- 📋 **Company policy** — Review your company's policies on AI tool usage before getting started
