# AI Agent 小手冊

> 給完全零基礎的同事 — 用 opencode 提高日常工作生產力

---

## Part 1：什麼是 AI Agent？

### 什麼是 AI Agent？

想像你有一位**超級實習生**。他：
- 聽得懂你說的話（自然語言）
- 可以動手幫你做事（不只是聊天）
- 會讓你看他做了什麼（你可以批准或拒絕）

**AI Agent = AI 大腦 + 動手做事的能力**

---

### 跟 ChatGPT 有什麼不同？

| | ChatGPT | opencode (AI Agent) |
|---|---|---|
| 做什麼 | 回答問題、聊天 | 動手幫你處理文件、整理資料、修改檔案 |
| 給你什麼 | 文字答案 | 實際成果（修改好的檔案、整理好的資料） |
| 互動方式 | 一問一答 | 你看著它做，可以中途叫它停或修改 |
| 例子 | 「幫我寫一封 email」→ 給你一段文字 | 「幫我寫一封 email」→ 幫你開好檔案寫完，給你看過，你同意就儲存 |

**簡單來說：ChatGPT 是「告訴你怎麼做」，opencode 是「幫你做好」**

---

### opencode 的三種使用方式

opencode 可以在三種環境中使用，選擇一種即可：

| 版本 | 適合誰 | 使用方式 |
|---|---|---|
| **Desktop App** 🖼️ | **一般使用者**（就是你） | 像一般應用程式一樣開啟，有圖形介面 |
| Terminal (TUI) ⌨️ | 開發者 | 命令列模式 |
| VS Code Extension 🧩 | 開發者 | 在 VS Code 編輯器中使用 |

> **Desktop App 是為你設計的** — 像開啟 Chrome、Word 一樣，雙擊圖示就能開啟。

---

### 重要概念

**1. Conversation（對話）**
你與 opencode 的每一次互動稱為一個 conversation。就像 WhatsApp 一樣，一句接一句。

**2. Context（背景資料）**
你可以提供檔案、圖片、網頁給 opencode，讓它了解你在說什麼。
> 「幫我修改這份文件」→ 你需要先把文件給它看

**3. Plan mode vs Build mode**
- **Plan mode** 📋：opencode 只會告訴你它打算怎麼做，不會實際動手
- **Build mode** 🔨：opencode 會實際執行

> 建議：不確定的時候先用 Plan mode，看過計劃再讓它執行。

**4. /undo（還原）**
如果 opencode 做了你不想要的修改，輸入 `/undo` 就會還原。不用擔心弄壞東西。

---

### 你需要準備什麼

| 你不需要會 | 你需要有 |
|---|---|
| ❌ 寫程式 | ✅ 一台電腦（Mac / Windows） |
| ❌ 使用命令列 | ✅ 網路連線 |
| ❌ 了解 API | ✅ 一個 opencode 帳號（免費註冊） |
| ❌ 了解 Terminal | ✅ 大概知道自己想做什麼 |

---

### 它能幫你做什麼？

| 日常工作 | 你對它說 | opencode 會做 |
|---|---|---|
| 寫 Email | 「幫我寫一封 email 給客戶確認會議時間」 | 寫好信件內容給你看 |
| 修改文章 | 「這段太長了，幫我縮短一半」 | 修改完成給你看 |
| 翻譯 | 「把這份中文翻譯成英文」 | 翻譯完成給你看 |
| 整理資料 | 「幫我整理這個 Excel，找出前 10 大客戶」 | 整理好後回傳檔案給你 |
| 研究調查 | 「幫我研究 competitor X 的定價策略」 | 上網蒐集資料，整理好給你 |

## Part 2：安裝與設定

### 第一步：安裝 opencode Desktop

#### Mac 使用者

**方法一：使用安裝腳本（推薦）**

開啟「終端機」（Terminal），貼上以下指令後按 Enter：

```bash
curl -fsSL https://opencode.ai/install | bash
```

> 如果出現密碼提示，請輸入你的電腦密碼（輸入時不會顯示字元，這是正常的）。

**方法二：手動下載**

1. 前往 [opencode.ai/download](https://opencode.ai/download)
2. 選擇 macOS 版本下載
3. 開啟下載的檔案，將 opencode 拖曳到「應用程式」資料夾

#### Windows 使用者

**方法一：使用安裝腳本（推薦）**

1. 在工作列搜尋「PowerShell」
2. 在「Windows PowerShell」上按右鍵，選擇「以系統管理員身分執行」
3. 貼上以下指令後按 Enter：

```powershell
winget install -e --id SST.OpenCodeDesktop
```

**方法二：手動下載**

1. 前往 [opencode.ai/download](https://opencode.ai/download)
2. 選擇 Windows 版本下載（.msi 檔案）
3. 雙擊安裝檔案，依照指示完成安裝

---

### 第二步：註冊並取得 API Key

1. 開啟 opencode Desktop
2. 你會看到一個歡迎畫面，選擇「Connect」
3. 瀏覽器會自動開啟 [opencode.ai/auth](https://opencode.ai/auth)
4. 登入或註冊一個新帳號
5. 選擇付費方案（有免費額度可以試用）
6. 複製畫面顯示的 API Key
7. 回到 opencode Desktop，貼上 API Key

> 💡 **什麼是 API Key？** 它就像一組密碼，讓 opencode 可以連接 AI 服務。你只需要設定一次。

---

### 第三步：確認安裝成功

1. 開啟 opencode Desktop
2. 你應該會看到一個對話視窗
3. 輸入「Hello」或「你好」，看看它是否會回覆你

如果它回覆了，恭喜你，安裝成功！🎉

## Part 3：基本操作

### 開啟一個對話

1. 開啟 opencode Desktop
2. 你會看到一個輸入框，像聊天軟體一樣
3. 在輸入框中打字，按 Enter 送出

---

### 提供背景資料

opencode 需要知道你在說什麼，才能給你好答案。

**提供檔案：**
- 直接把檔案拖曳到對話視窗
- 或點擊輸入框旁邊的「+」按鈕選擇檔案
- 支援的檔案類型：文字檔（.txt）、PDF、Word（.docx）、Excel（.xlsx）、圖片等

**提供圖片：**
- 直接把圖片拖曳到對話視窗
- opencode 可以「看」圖片內容

**提供網頁連結：**
- 直接貼上網址，opencode 會自動讀取網頁內容

---

### Plan mode 與 Build mode

這是 opencode 最重要的功能之一。

**Plan mode（計劃模式）📋**
- opencode 只會告訴你「它打算怎麼做」
- 不會實際修改任何檔案
- 適合：不確定要怎麼做時，先看看計劃

**如何切換：**
- 在輸入框下方有一個切換按鈕
- 點擊可以切換 Plan mode ↔ Build mode
- 你也可以直接告訴它：「先給我一個計劃」

**Build mode（執行模式）🔨**
- opencode 會實際動手執行
- 會修改檔案、建立檔案、整理資料
- 適合：你已經確定要做什麼

---

### 常用指令

| 指令 | 作用 |
|---|---|
| `/undo` | 還原上一次的修改 |
| `/redo` | 重做上一次還原的修改 |
| `/clear` | 清除當前對話記錄 |
| `/help` | 顯示所有可用指令 |

---

### 如何給出好指令

好的指令 = 好的結果。以下是幾個原則：

**1. 說清楚你要什麼**
- ❌ 「幫我改一下這個」
- ✅ 「幫我把這段文字改得更正式，適合寫給客戶」

**2. 提供足夠背景**
- ❌ 「幫我寫一封 email」
- ✅ 「幫我寫一封 email 給客戶張先生，確認下週一上午 10 點的會議，附上會議連結」

**3. 一次做一件事**
- ❌ 「幫我改這個檔案，然後寫一封 email，再整理那個 Excel」
- ✅ 分三次問，一次一件事

**4. 不滿意就說**
- ❌ （默默自己改）
- ✅ 「這個版本太長了，幫我縮短到 200 字以內」

## Part 4：日常應用場景

### 場景一：寫 Email

**你對它說：**
```
幫我寫一封 email 給客戶王小姐，內容包括：
1. 感謝她上週參加我們的產品說明會
2. 附上產品目錄 PDF
3. 詢問她是否有興趣安排一次一對一示範
4. 語氣要專業但友善
```

**opencode 會做：**
- 寫好完整的 email 內容給你看
- 你可以要求修改語氣、增減內容
- 滿意後直接複製貼到 Outlook/Gmail

---

### 場景二：修改文章

**你對它說：**
```
這段文字太囉嗦了，幫我濃縮成 100 字以內：

[貼上你的文字]
```

**或者：**
```
幫我把這段文字改成更正式的商業語氣：

[貼上你的文字]
```

---

### 場景三：翻譯

**你對它說：**
```
把以下內容翻譯成繁體中文：

[貼上英文內容]
```

**或者：**
```
幫我把這份文件翻譯成英文，保留原本的格式：

[貼上文件或拖曳檔案]
```

---

### 場景四：整理資料

**你對它說：**
```
我有一個 Excel 檔案，裡面是本月客戶訂單資料。
幫我：
1. 找出消費金額最高的前 10 名客戶
2. 計算每位客戶的平均訂單金額
3. 整理成一個表格

[拖曳 Excel 檔案到對話視窗]
```

---

### 場景五：摘要重點

**你對它說：**
```
幫我 summarize 這篇文章，列出 5 個重點：

[貼上文章內容或網址]
```

---

### 場景六：腦力激盪

**你對它說：**
```
我們下個月要辦一場線上研討會，主題是「AI 如何幫助中小企業」。
幫我想 10 個吸引人的活動名稱，以及 3 種不同的活動形式。
```

---

### 場景七：研究調查

**你對它說：**
```
幫我研究 competitor X 這家公司：
1. 他們的主要產品是什麼？
2. 定價大約多少？
3. 他們的目標客戶是誰？
4. 跟我們相比，優缺點是什麼？
```

> 💡 opencode 可以上網搜尋資料，但建議你還是要自己驗證資訊的正確性。

## Part 5：實用小貼士

### 如何問得更好

**1. 從簡單開始**
第一次使用時，先問簡單的問題，熟悉它的回覆方式。

**2. 逐步深入**
先問大方向，再追問細節。就像剝洋蔥，一層一層來。

**3. 給它角色**
```
「你是一位專業的業務助理，幫我寫一封...」
「你是一位行銷顧問，幫我分析...」
```
給它一個角色，它會用更符合該角色的語氣回覆。

**4. 指定格式**
```
「請用條列式回答」
「請用表格呈現」
「請用 3 段話說明」
```

---

### 不滿意怎麼辦？

1. **直接說** — 「這個不行，因為...」然後告訴它要改什麼
2. **給範例** — 「像這樣：...」給它一個參考
3. **重新來過** — 輸入 `/clear` 清除對話，重新問一次
4. **還原修改** — 如果它改了檔案但你不滿意，輸入 `/undo`

---

### 安全提醒

- **不要上傳機密資料** — 如果你不確定資料是否敏感，先詢問主管
- **驗證重要資訊** — AI 有時會出錯，重要資料請自己再確認
- **公司政策** — 使用前請確認公司對於使用 AI 工具的相關規定

## Part 6：常見問題

### Q：我完全不會寫程式，可以用嗎？
**A：可以。** opencode Desktop 有圖形介面，像一般應用程式一樣操作。你只需要會打字就可以了。

### Q：需要付費嗎？
**A：opencode 本身是免費的開源軟體，但你使用的 AI 模型需要付費。opencode 提供付費方案（有免費額度），也可以連接其他 AI 服務。

### Q：我的資料安全嗎？
**A：這取決於你選擇的 AI 服務商。opencode 本身不會儲存你的資料。建議不要上傳公司機密資料，並遵守公司的資訊安全政策。

### Q：AI 會出錯嗎？
**A：會。** AI 不是完美的，它可能會：
- 理解錯誤
- 給出錯誤的資訊
- 產生不合理的內容

**所以重要的事情一定要自己再確認一遍。**

### Q：opencode 跟 ChatGPT 有什麼不同？
**A：ChatGPT 是「告訴你怎麼做」，opencode 是「幫你做好」。** opencode 可以直接修改檔案、整理資料、執行動作，而不只是給你建議。

### Q：遇到問題可以去哪裡問？
**A：**
- opencode 官方文件：[opencode.ai/docs](https://opencode.ai/docs)
- opencode GitHub：[github.com/anomalyco/opencode](https://github.com/anomalyco/opencode)
- 詢問已經在使用的同事

---

> **最後提醒：AI 是工具，不是萬能的。把它當成一個很聰明的助手，但最終的判斷還是要靠你自己。**
